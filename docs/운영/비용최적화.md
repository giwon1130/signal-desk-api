# 비용 최적화 (백엔드 SoT)

> **누가 봄**: 운영자
> **언제 봄**: Railway 청구서 비싸졌을 때
> **공유**: app 저장소 [비용최적화](https://github.com/giwon1130/signal-desk-app/tree/main/docs/운영/비용최적화.md) 와 동일 — 백엔드 구현 디테일 보충.

## 적용 완료

### 1. WS 폴링 휴장 가드
(stub) `PriceTickerHandler.broadcastPrices`
- 매 5초 실행 시 `marketSessionService.buildMarketSessions()` 호출 (cheap, 메모리만)
- KR/US OPEN 여부 체크 → fetch 자체 skip
- 양쪽 휴장이면 외부 API 호출 0
- **효과**: 24시간 중 ~13시간만 활성 → 외부 호출 50~65% ↓

### 2. Cache TTL 동적
(stub) `MarketOverviewService.coreTtl()` / `newsTtl()`
- OPEN 중: core 60s / news 5m
- 양쪽 휴장: core 3m / news 15m
- **효과**: 휴장 시간대 외부 호출 ~3배 ↓

### 3. CORS 좁힘
운영 도메인만 허용 → 외부 봇 차단

### 4. IP rate limit
- /auth 5/분
- /api/v1/market 60/분
- 봇 트래픽 차단

### 5. JVM 메모리
- `-XX:MaxRAMPercentage=75.0` — 컨테이너 한도의 75% 사용
- `-XX:+UseG1GC` — 메모리 효율 좋음
- `-XX:+ExitOnOutOfMemoryError` — OOM 시 빨리 죽고 재시작 (실패 누적 방지)

### 6. Tomcat threads
- max 50 (default 200 → 다운). 트래픽 작은 단계 OK.

### 7. Hikari pool
- max 5 (default 10 → 다운). 충분.

## 추가 절감 후보

(stub)
- **/api/v1/market/* 인증 게이트** — 익명 호출 차단. 사용자 늘면 검토.
