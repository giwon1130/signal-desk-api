## JWT (`features/auth/application/JwtProvider.kt`)

(stub)
- 알고리즘: HS256
- 키: `Keys.hmacShaKeyFor(secret.padEnd(32, 'x').toByteArray())`
  - secret 짧으면 padding 으로 32바이트 채움 (취약 — secret 길이 검증 강화 필요)
- 만료: env `SIGNAL_DESK_JWT_EXPIRATION_HOURS` (기본 720h)
- claims: `sub` (UUID) + `email`
- 만료 시 `parseSignedClaims` throw → controller 에서 401

### 부팅 시 secret 검증
- 더미값(`change-me-please-...`) → ERROR 로그
- 32자 미만 → WARN 로그
- 자동 fail-fast 안 함 (Railway healthcheck 끊겨서 디버깅 불가)

## OAuth (`features/auth/application/AuthService.kt`)

### Google
1. 클라 idToken → 서버
2. 서버: `https://oauth2.googleapis.com/tokeninfo?id_token=...` 호출
3. **audience 검증**: `signal-desk.auth.google.allowed-audiences` 에 `aud` 가 있어야 통과
4. 비어있으면 검증 스킵 + WARN (개발 호환)
5. `sub` 로 사용자 매칭/생성

### Kakao
- `https://kapi.kakao.com/v2/user/me` 호출
- audience 검증 없음 (Kakao 는 `aud` 개념이 다름)

## Rate Limit (`bootstrap/RateLimitFilter.kt`)

(stub)
- Bucket4j in-memory token bucket per IP
- IP 식별: `X-Forwarded-For` 첫 토큰 (Railway proxy) → fallback `remoteAddr`
- key: `{rule}:{ip}` (예: `auth:1.2.3.4`)
- 1만 키 초과 시 단순 비움 (메모리 누수 방지 — 휴리스틱)

### 알려진 한계
- 단일 인스턴스 메모리 — scale-out 시 IP limit 인스턴스마다 따로 (현재 1 인스턴스 OK)
- 시간 만료 없음 — size-based 청소만
- IP 위조: `X-Forwarded-For` 신뢰. Railway 가 채워준다고 전제.
