# 환경변수 (백엔드)

> **누가 봄**: 운영자
> **언제 봄**: 새 환경 셋업 / 변수 추가

## Spring Boot @Value 매핑

Spring 은 `signal-desk.foo.bar` ↔ `SIGNAL_DESK_FOO_BAR` 자동 매핑.
실제 application.yml 의 키는 dot 표기, 환경변수는 SCREAMING_SNAKE.

## 필수

(stub)
| 키 (yml) | 환경변수 | 의미 | 운영 예시 |
|---------|---------|------|----------|
| `signal-desk.store.mode` | `SIGNAL_DESK_STORE_MODE` | 저장 모드 | `jdbc` |
| | `JDBC_DATABASE_URL` 또는 `DATABASE_URL` | Postgres URL | Railway 자동 |
| `signal-desk.jwt.secret` | `SIGNAL_DESK_JWT_SECRET` | JWT 서명 키 (32자+) | `openssl rand -base64 48` |
| `signal-desk.auth.google.allowed-audiences` | `SIGNAL_DESK_AUTH_GOOGLE_ALLOWED_AUDIENCES` | Google audience 화이트리스트 | 콤마 구분 클라이언트 ID |

## 부팅 시 검증

(stub)
- JWT secret 더미값 → `[SECURITY] ERROR` 로그
- JWT secret 32자 미만 → `[SECURITY] WARN` 로그
- Google audiences 미설정 → `[SECURITY] WARN` 로그 (검증 스킵)

## secret rotation

(stub)
- JWT: 새 값 → env 갱신 → 자동 재시작 → 모든 사용자 재로그인 필요
- Google audience: 새 ID 추가 → env 콤마로 추가 → 재시작
