# 의사결정 로그 (ADR — 백엔드)

> **누가 봄**: 백엔드 개발자
> **언제 봄**: 큰 결정 직전 (전례 검색) / 결정 직후 (기록)
> **공유**: 일부 결정은 클라이언트 ADR 와 중복. 백엔드 디테일 위주.

## 형식

```
## ADR-XXX — YYYY-MM-DD
**결정**: 한 줄
**대안**: 검토했던 다른 옵션들
**선택 이유**: 왜 이걸 골랐나
**영향**: 다른 코드/설정에 미치는 영향
**되돌리기 비용**: 낮음 / 중간 / 높음
**관련**: PR / 이슈
```

## ADR-015 — 2026-05-29
**결정**: KR 마감 브리프(15:40 KST) 기본 ON 전환 (V23). 모닝 ON / 장중·이브닝 OFF 유지.
**대안**: 전부 opt-in 유지, 전부 기본 ON
**선택 이유**: 토글 UI 미배포 상태라 의도적으로 끈 사용자가 없음 → 기존 행 일괄 ON + 컬럼 default true. 마감 브리프는 하루 정리용으로 가치 높아 우선 노출. 장중·이브닝은 알림 피로도 고려해 opt-in 유지.
**영향**: V22 에서 `midday_brief_enabled`/`close_brief_enabled` false 로 추가 → V23 에서 close 만 default true + 기존 행 update.
**되돌리기 비용**: 낮음
**관련**: 마이그 V22 / V23

## ADR-014 — 2026-05-29
**결정**: 리딩(애널리스트 콜) 도입 — `entry_price` immutable 박제 (V21).
**대안**: 콜 가격을 실시간 재계산, 작성자가 수정 가능
**선택 이유**: 콜의 신뢰성은 "작성 시점 시세를 못 바꾼다"는 데서 옴. `entry_price`/`entry_currency`/`entry_locked_at` 을 작성 시 박제하고 UPDATE 금지. 성과(수익률·적중)는 현재가 대비 derive.
**영향**: `reading_call` 테이블, `ReadingService.publishPost` 가 게시 시점 시세 fetch + 박제. 4테이블(leader/follow/post/call). jdbc 모드 전용.
**되돌리기 비용**: 중간 (테이블·API 추가)
**관련**: 마이그 V21, spec `leading-call-spec.md`

## ADR-013 — 2026-05-29
**결정**: `push_alert_log.direction` varchar(8) → varchar(20) 확장 (V20).
**대안**: enum 값 약어로 줄이기
**선택 이유**: `PRICE_ABOVE`/`PRICE_BELOW`(11)·`VOLUME_SPIKE`(12) 가 varchar(8) 초과 → INSERT "value too long" 예외 → `recordAlert` 실패 → dedup 무력화 → 매 5분 재발송 스팸. 컬럼 폭만 늘려 모든 `AlertDirection` enum 수용.
**영향**: 알림 dedup 정상화. 데이터 손실 없음.
**되돌리기 비용**: 낮음
**관련**: 마이그 V20

## ADR-012 — 2026-05-27
**결정**: 모의투자 리그 — immutable trade 원장 + non-stored derived position (V19).
**대안**: position 테이블 저장 후 매 거래마다 UPDATE, 페이퍼 트레이딩 재도입
**선택 이유**: trade 를 append-only 로 두면 정정·감사·리액션이 단순. position 은 trade 합산으로 derive (DB 미저장)해 정합성 깨질 여지 제거. 다국적 league 위해 거래 시점 환율 lock(`exchange_rate`/`price_locked_at`).
**영향**: 4테이블(league/participant/trade/reaction). 코드에서 trade UPDATE/DELETE 금지. `PositionService` 가 런타임 계산. jdbc 모드 전용. (V13 에서 제거한 페이퍼 트레이딩의 후속 대체)
**되돌리기 비용**: 중간
**관련**: 마이그 V19, spec `mock-investment-game-spec.md`

## ADR-009 — 2026-05-15
**결정**: Gemini 기반 미디어 브리프 + 뉴스 종합, 스케줄 생성 + 캐시.
**대안**: 룰 기반 요약, 외부 요약 API

**영향**: `media` 패키지, `GeminiClient`, 스케줄러. `GET /api/v1/media/*`, `/api/v1/insights/today`. `SystemStatusController` 가 Gemini 헬스 노출.
**되돌리기 비용**: 중간
**관련**: 마이그 V10 / V11
