## 외부 client 작성 패턴

(stub)
- `HttpClient.newBuilder().connectTimeout(3s).build()` (짧은 timeout)
- 요청에도 `.timeout(5s)` 명시
- `runCatching { ... }.getOrNull()` 또는 `.getOrElse { default }`
- 실패 시 `logger.warn` (silent fallback 금지)
- 응답 파싱 실패도 같이 catch

## 응답 wrapper

```kotlin
data class ApiResponse<T>(val success: Boolean, val data: T? = null, val error: String? = null)
```

성공: `ApiResponse(true, value)`
실패: `@RestControllerAdvice` 가 `ApiResponse(false, null, exception.message)` 로 변환
